[使い方]

入力ファイルは AVI ファイルか、複数の PNG ファイルのどちらかになる。
32ビット無圧縮 AVI ファイルは、には 32ビット無圧縮形式のものを指定する。
他の形式については保証しない。
連番 PNG フォルダは、複数の PNG ファイルが入ったフォルダを指定する。
動画のサイズは最初の PNG ファイルの大きさに合わせられる。
異なる画像サイズのファイルがあった場合の動作は未保証。
連番と書いてあるが、実際は名前順でソートされるだけなので、別に連番でなくても良い。

出力ファイル名には出力するファイル名を指定する。

FPS は、FPS。再生速度を指定する。
AVI ファイルの場合は、AVI ファイルから値が設定される。
PNG の時は、自ら指定する。
Scale / Rate (秒)が、1枚あたりの表示時間。
小数点以下の値を持つ FPS 値を指定する必要がないのであれば、Scale を 1 にしておけば、Rate がそのまま FPS 値となる。

アルファチャンネル圧縮方式は、アルファチャンネルをどのように圧縮するかを選択できる。
可逆圧縮と非可逆圧縮が選択できる。
立ち絵などの不透明と完全透明が画像のほとんどの領域を占める場合、可逆圧縮を選択した方が圧縮率は高い。
中間階調を多く含む場合は非可逆圧縮の方が良い傾向を示す。

入力アルファ形式は、通常の形式(ストレート)か Add Alpha 形式かを選択する。
なんらかのソフトでレンダリングした時、プリマルチプライ形式(背景は黒)を指定した場合、Add Alpha 形式を選択してやればうまくいくと思われます。
レンダリング時にストレートが選択できるのあれば、ストレートでレンダリングし通常形式を選択した方が確実です。

クオリティーでは、画像のクオリティーを指定します。
クオリティーの値を大きくすれば出力画像は綺麗になりますが、圧縮率は低下します。
逆にクオリティーの値を小さくすれば、出力画像は大きく劣化しますが、圧縮率は向上します。
立ち絵などのくっきりした画像の場合、クオリティーを下げると劣化が目立つためあまり下げることは出来ませんが、エフェクトのような形のはっきりしない画像の場合はクオリティーを下げても違いはわかり辛いです。
動画ごとにどの辺りまで下げても問題なさそうか調整することで、ファイルサイズを抑えることが出来ます。
完全透明ブロックの維持では、完全に透明となる領域を可逆可能にするかどうかを選択できます。
完全透明な領域に薄く色が出ているような時は、このオプションを指定します。
基本的には、常にこのオプションは選択状態の方が良いです。
ただし、このオプションが有効な場合は少し圧縮率が低下します。
このオプションをはずしても問題なさそうな場合は、解除することでファイルサイズを少し抑えることが出来ます。

上記オプションを指定して「開始」を選択するとエンコードが開始されます。
エンコード対象はキューイングされるので、エンコード中も次の動画のエンコードの設定が出来、次々にエンコードを開始していくことが出来ます。
